大模型推理入门学习3：常用的集合通信原语简介

在分布式训练或推理中，集合通信（Collective Communication）是实现GPU设备间数据交互的关键操作，具体地它是指：在通信组（group）中，由所有进程或节点（rank）共同参与的一种通信操作。各rank通过发送、接收、复制、同步等行为，完成特定的通信模板（或称通信原语、通信拓扑）。常见操作包括broadcast、gather、all-gather、scatter、reduce、all-reduce、reduce-scatter、all-to-all等。需要注意的是，通常情况下，我们将单个设备（如单张GPU）理解为一个rank，但实际上单个rank也可以包含多个GPU。

在集合通信中，数据维度的明确性至关重要。对于同一种集合通信操作，若处理的数据维度不同，其对输入数据的要求以及最终输出结果也会随之改变。本文主要面向LLM模型场景，输入激活的形状一般是(batchsize,seq_len,hidden_size)，若分别对序列维度（seq_len）和隐藏维度（hidden_size）的数据执行聚合操作，这两种场景下所需输入数据的尺寸（维度）存在显著差异。如下图所示，为便于描述，可以将这两种操作分别理解为数据的行（row）聚合或列（column）聚合。

一、BroadCast：广播

broadcast的含义是：一个rank拥有数据，把这份数据复制给所有其它rank上

通信前：

rank0: X
rank1: empty
rank2: empty
rank3: empty

通信后：

rank0: X
rank1: X
rank2: X
rank3: X

这是一种一对多的同步，它的典型的应用场景有：

  1. 数据并行的参数初始化，确保每张卡上的初始参数是一致的
  2. allReduce中的一个步骤

二、gather：收集

gather的含义是：每个 rank 都有一份数据，把所有 rank 的数据收集到某一个目标 rank 上。，包含row gather和column gather

以row gather为例，通信前：
通信前：

rank0: [[1, 2]]
rank1: [[3, 4]]
rank2: [[5, 6]]
rank3: [[7, 8]]

Gather 到 rank0 后：

rank0:[[1, 2], [3, 4], [5, 6], [7, 8]]
rank1:
rank2:
rank3:

应用场景有：
1.日志收集

三、All-Gather：全收集

all-gather的含义是：每个 rank 都有一份局部数据，通信后所有 rank 都拿到完整拼接结果。，也是包含row/column两种情况，但此处为了简单不再专门说明。

通信前：

rank0: X0
rank1: X1
rank2: X2
rank3: X3

All-Gather 后：

rank0: [X0, X1, X2, X3]
rank1: [X0, X1, X2, X3]
rank2: [X0, X1, X2, X3]
rank3: [X0, X1, X2, X3]

应用场景：
1.TP中：
列并行 Linear 后，每张 GPU 只有一部分输出：

rank0: Y0 [B,S,H/4]
rank1: Y1 [B,S,H/4]
rank2: Y2 [B,S,H/4]
rank3: Y3 [B,S,H/4]

如果后续算子需要完整 hidden，就要 All-Gather：

all-rank：Y [B,S,H]

2.SP中：
每张 GPU 只有部分 token：

rank0: [B,S/4,H]
rank1: [B,S/4,H]
rank2: [B,S/4,H]
rank3: [B,S/4,H]

如果某个算子需要完整序列，则 All-Gather 成：

all-rank：[B,S,H]

四、scatter：分发

scatter的含义是：一个 rank 拥有完整数据，把它切分后分发给不同 rank。 其实就是gather的反向操作。

通信前：

rank0: [X0, X1, X2, X3]
rank1: empty
rank2: empty
rank3: empty

Scatter 后：

rank0: X0
rank1: X1
rank2: X2
rank3: X3

在实际场景中，更常用的时reduce-scatter，而不是单独的scatter，单独的可以用来：

  1. 把输入的batch分发给不同GPU

五、reduce：规约

reduce的含义是：多个 rank 上的数据做某种归约运算，结果只放到一个目标 rank 上。 最常见的归约运算就是求和

通信前：

rank0: X0
rank1: X1
rank2: X2
rank3: X3

Reduce 到 rank0 后：

rank0: X0 + X1 + X2 + X3
rank1: X1
rank2: X2
rank3: X3

六、all-reduce：全规约

相当于reduce+broadcast：

通信前：

rank0: X0
rank1: X1
rank2: X2
rank3: X3

All-Reduce sum 后：

rank0: X0 + X1 + X2 + X3
rank1: X0 + X1 + X2 + X3
rank2: X0 + X1 + X2 + X3
rank3: X0 + X1 + X2 + X3

典型应用场景：
1.在TP（行切分）之后，如果后续的某个步骤需要每个rank拥有全部数据，就可以在该步骤之前做一次all-reduce

七、reduce-scatter：规约并分发

可以理解为先reduce再scatter，同样分别行列两个维度版本，此处以row reduce scatter为例

通信前：

rank0:[[1, 1], [2, 2]]
rank1:[[3, 3], [4, 4]]

执行 Row Reduce-Scatter（sum）后：

rank0:[[4, 4]]
rank1:[[6, 6]]

应用场景：既可以用于数据并行也可以用于模型并行

八、alltoall：全交换

每个 rank 都把自己的数据切成多份，分别发送给所有 rank；同时也从所有 rank 接收对应分片

通信前：

rank0: [A0, A1, A2, A3]
rank1: [B0, B1, B2, B3]
rank2: [C0, C1, C2, C3]
rank3: [D0, D1, D2, D3]

All-to-All 后：

rank0: [A0, B0, C0, D0]
rank1: [A1, B1, C1, D1]
rank2: [A2, B2, C2, D2]
rank3: [A3, B3, C3, D3]

应用场景：1.MOE；2.分布式张量布局转换或分布式矩阵转置
